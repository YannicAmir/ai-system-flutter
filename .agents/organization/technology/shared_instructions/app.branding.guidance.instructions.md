# App Branding Guidance

> When brand assets change, **both** the native splash and the Flutter splash must be updated. They are independent layers and neither substitutes for the other.

---

## 1. Overview

| Concept | What it is | Managed by |
|---|---|---|
| **Native splash** | What the OS shows instantly on cold launch, before Flutter renders | Native platform files (Storyboard / XML) |
| **Flutter splash** | The animated brand screen rendered by Flutter after the engine loads | Flutter widget (`AppLogo` / `AppLandingPage`) |
| **App icon** | The icon shown on the home screen / launcher | `flutter_launcher_icons` (iOS) + manual (Android) |

## 2. App Icon

### Source Files

```
assets/icons/launcher/
├── launcher-icon.png              ← source for iOS (all sizes generated from this)
└── launcher-icon-foreground.png   ← foreground layer for Android adaptive icon
```

### iOS — flutter_launcher_icons

iOS icons are generated from `launcher-icon.png` using the `flutter_launcher_icons` package. Configuration lives at the bottom of `pubspec.yaml`:

```yaml
flutter_launcher_icons:
  android: false          # Android icons are managed manually
  ios: true
  remove_alpha_ios: true  # required — Apple rejects icons with alpha channel
  image_path: 'assets/icons/launcher/launcher-icon.png'
```

After replacing the source PNG, regenerate with:
```
dart run flutter_launcher_icons
```

This writes the generated icon set into `AppIcon.appiconset`. Do **not** edit the generated files manually.

### Android — Manual

Android icons are **not** generated by `flutter_launcher_icons` (`android: false`). They are managed manually:

```
android/app/src/main/res/
├── mipmap-mdpi/        ← ic_launcher.png + ic_launcher_foreground.png
├── mipmap-hdpi/
├── mipmap-xhdpi/
├── mipmap-xxhdpi/
├── mipmap-xxxhdpi/
└── mipmap-anydpi-v26/
    └── ic_launcher.xml  ← adaptive icon definition
```

The adaptive icon composes a background colour with the foreground PNG:

```xml
<adaptive-icon>
  <background android:drawable="@color/ic_launcher_background"/>
  <foreground android:drawable="@mipmap/ic_launcher_foreground"/>
</adaptive-icon>
```

To update the Android icon: replace `ic_launcher_foreground.png` at every density and update the background colour in `res/values/` if needed.

## 3. Native Splash Screen

The native splash is what the OS displays on cold start, before Flutter has initialised. No Flutter code is involved.

### iOS — LaunchScreen.storyboard

Located at `LaunchScreen.storyboard`. Current implementation is a plain coloured background (brand primary red) with no image — intentionally minimal to avoid a jarring flash when the Flutter splash appears.

```xml
<color key="backgroundColor"
  red="0.90588..."
  green="0.07450..."
  blue="0.14117..."   <!-- brand primary #E71324 in normalised RGB -->
  alpha="1"/>
```

`Assets.xcassets` also contains a `LaunchImage.imageset` with `@1x`/`@2x`/`@3x` slots — currently blank/transparent. Populate these if the splash needs a logo.

**To update the iOS native splash:**
- Background colour: edit the `<color>` value in `LaunchScreen.storyboard`
- Add a logo: replace the PNG files in `LaunchImage.imageset` at all three scales

### Android — launch_background.xml

Located at `launch_background.xml`. Currently renders a single background image filling the screen:

```xml
<layer-list>
  <item>
    <bitmap android:gravity="fill" android:src="@drawable/background"/>
  </item>
</layer-list>
```

The source bitmap is `background.png` in `res/drawable/`.

**To update the Android native splash:**
- Replace `background.png` in `res/drawable/`
- To add a centred logo over a colour, extend `launch_background.xml` with a second `<item>` containing a centred `<bitmap>`

## 4. Flutter Splash Screen

After the native splash, Flutter renders its own animated splash via the `AppLogo` widget at the `/applanding` route.

```
assets/icons/splash/
├── tkmaxx-splash-logo.png    ← the brand wordmark/logo
└── eclipse.png               ← decorative ring/border overlay
```

```dart
// AppLogo widget composition
Container(
  decoration: ShapeDecoration(shape: CircleBorder(), color: AppColors.primary),
  height: 270,
  width: 270,
  child: Stack(children: [
    Image.asset(AppIconPaths.splashlogo),   // wordmark
    Image.asset(AppIconPaths.eclipse),      // ring overlay
  ]),
)
```

**To update the Flutter splash:** replace the asset files in `assets/icons/splash/`. Paths are referenced via `AppIconPaths.splashlogo` and `AppIconPaths.eclipse` — never hardcode splash asset paths.

## 5. Rules

| Rule | Detail |
|---|---|
| App icon source | Single PNG in `assets/icons/launcher/` |
| iOS icon generation | `flutter_launcher_icons` — run `dart run flutter_launcher_icons` after changing source |
| Android icon | Manually managed in `android/app/src/main/res/mipmap-*/` — `flutter_launcher_icons` NOT used |
| Native splash (iOS) | `LaunchScreen.storyboard` — plain brand colour background |
| Native splash (Android) | `launch_background.xml` + `background.png` in `res/drawable/` |
| Flutter splash | `AppLogo` widget using `AppIconPaths` constants — NEVER hardcode splash asset paths |
| Brand change | Update BOTH native splash AND Flutter splash assets |
| iOS alpha channel | `remove_alpha_ios: true` MUST remain in `pubspec.yaml` config |

---

## Checklist
- [ ] New brand asset file(s) placed in correct `assets/icons/` subfolder
- [ ] `AppIconPaths` updated with any new or changed path getters
- [ ] If app icon changed: `launcher-icon.png` and/or `launcher-icon-foreground.png` replaced in `assets/icons/launcher/`
- [ ] If iOS app icon changed: `dart run flutter_launcher_icons` run — generated files not manually edited
- [ ] If Android app icon changed: `ic_launcher_foreground.png` replaced at every mipmap density
- [ ] If iOS native splash changed: `LaunchScreen.storyboard` colour value updated and/or `LaunchImage.imageset` PNGs replaced
- [ ] If Android native splash changed: `background.png` replaced in `res/drawable/` and/or `launch_background.xml` updated
- [ ] If Flutter splash changed: asset files in `assets/icons/splash/` replaced — no path strings hardcoded
- [ ] `remove_alpha_ios: true` still present in `pubspec.yaml`
- [ ] Both native splash AND Flutter splash updated when brand changes require it
